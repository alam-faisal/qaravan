## What was accomplished

### New public API

**`core/base.py`**
- `State.partial_overlap(other, skip) -> np.ndarray` — non-abstract with `NotImplementedError` default (same pattern as `NoiseModel.get_kraus`). Convention: `result[i,j] = ⟨other[j]|self[i]⟩`.
- `State.overlap(other) -> complex` — promoted to a **concrete** base-class method: `return self.partial_overlap(other, skip=[])[0,0].conj()`. Backends no longer need to override `overlap`; implementing `partial_overlap` is sufficient.

**`backends/statevector.py`**
- `Statevector.partial_overlap(other, skip)` — one-line wrapper around the module-level function. The existing `Statevector.overlap` override was removed (covered by base class).

**`applications/circuit_library.py`**
- `brickwall_skeleton(Lx, Ly=1) -> list[list[int]]` — 2D brickwall in even-x horiz / odd-x horiz / even-y vert / odd-y vert order. Port of the Julia reference implementation. For `Ly=1` reduces to standard 1D brickwall. Depth via list repetition: `brickwall_skeleton(n) * d`.
- `two_local_circuit(skeleton, seed=None) -> Circuit` — Haar-random 4×4 unitaries on each pair in skeleton. Global reproducibility via per-gate seeds drawn from a top-level RNG.

**`applications/run_context.py`** (new file)
- `RunContext` — dataclass with `max_iter`, `stop_ratio`, `stop_absolute`, `quiet`. `should_stop(sweep_costs, sweep) -> bool` receives per-sweep costs for unambiguous convergence checking.

**`applications/compilation.py`** (new file)
- `environment_state_prep(target, init_state, circuit=None, skeleton=None, context=None) -> tuple[Circuit, list[float]]` — backend-agnostic environment sweep. Records infidelity after every gate update for smooth convergence curves. Input circuit is never mutated (copy at top).
- `_environment_update(circ, k, pre_state, post_state, init_state, target, direction) -> tuple[float, State, State]` — SVD polar-factor update; incremental pre/post state maintenance via `state.apply(Circuit([gate]))`.

**`applications/__init__.py`** — exposes all symbols above.

## Design decisions taken during implementation

**`partial_overlap` convention**: `result[i,j] = ⟨other[j]|self[i]⟩` (sv1=self in the module-level call). This gives `[0,0] = ⟨other|self⟩` — the conjugate of `⟨self|other⟩` — so `overlap` calls `.conj()`. This convention is consistent with the existing `rdm(sites)` implementation (`partial_overlap(flat, flat, skip=sites)`).

**`overlap` base-class concrete method**: calling `self.partial_overlap(other, skip=[])[0,0].conj()`. Verified by a test (`test_state_overlap_derived_from_partial_overlap`) that a State implementing only `partial_overlap` gets `overlap` for free.
