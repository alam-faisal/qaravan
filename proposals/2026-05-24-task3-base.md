# Task 3 â€” Write `core/base.py`

## Design overview

Six classes in one file. `Gate` and `Circuit` are **concrete** (no ABC).
`State`, `Simulator`, `Observable`, `NoiseModel` are **abstract** (`ABC`).
`Simulator.run()` is the one method defined in the base that calls abstract hooks.

---

## Class-by-class signatures

### `Gate`

```python
class Gate:
    def __init__(self, name: str, indices: int | list[int], matrix: np.ndarray, time: float | None = None)
    # indices normalised to list[int] in __init__
    # local_dim computed once in __init__ from matrix shape
    # time is usually set at compile time by Simulator; only meaningful for noise-aware backends

    @property
    def num_sites(self) -> int   # len(self.indices)
    self.local_dim: int          # matrix.shape[0] ** (1/num_sites), rounded, set in __init__

    def dagger(self) -> Gate     # conjugate-transpose matrix, append "â€ " to name
    def __str__(self) -> str
```

`time` is retained (some noise models need gate duration). Non-unitary operations
(`Reset`, `Measure`) subclass `Gate` in later tasks â€” no type-level distinction here.
`as_superop()` is **not** on `Gate` â€” superoperator conversion is done at compile time
by `Simulator`, which knows the backend context.

### `Circuit`

```python
class Circuit:
    def __init__(self, gates: list[Gate], n: int | None = None, local_dim: int = 2)
    # n inferred from max(gate.indices)+1 if gates non-empty and n is None

    self.gates: list[Gate]
    self.n: int
    self.local_dim: int
    self.layers: list[list[Gate]] | None   # None until construct_layers() called

    def construct_layers(self) -> None     # greedy layer packing; mutates self.layers
    def decompose(self, basis: str = 'ZSX') -> None  # expand U-gates; mutates self.gates
    def dagger(self) -> Circuit            # reversed gate-daggered copy (immutable)
    def to_matrix(self) -> np.ndarray      # full unitary (requires square gates)
    def copy(self) -> Circuit              # shallow-copy gate list, propagate layers/n

    def __add__(self, other: Circuit) -> Circuit   # concatenation
    def __mul__(self, n: int) -> Circuit           # repetition (n >= 0)
    def __rmul__(self, n: int) -> Circuit
    def __len__(self) -> int
    def __getitem__(self, key: int | slice) -> Gate | Circuit
    def __str__(self) -> str
```

`meas_sites` is **gone** (it was v0.1). `add_noise` / `build` are also gone â€” noise
insertion is the Simulator's job. `construct_layers` and `decompose` are the only
circuit-mutation methods.

### `State` (ABC)

```python
class State(ABC):
    @abstractmethod
    def expectation(self, observable: Observable) -> complex: ...

    @abstractmethod
    def sample(self, shots: int) -> np.ndarray: ...

    @abstractmethod
    def sample_and_collapse(self, sites: list[int]) -> tuple[str, State]: ...

    @abstractmethod
    def overlap(self, other: State) -> complex: ...
```

`rdm()` is **not** in the ABC â€” not every backend can compute reduced density matrices
efficiently (e.g. stabilizer states). Subclasses that support it add it themselves.
Concrete subclasses live in `backends/`.

---

### `Observable` (ABC)

```python
class Observable(ABC):
    def __init__(self, name: str, indices: int | list[int])
    # indices normalised to list[int] in __init__

    self.name: str
    self.indices: list[int]
```

No methods on the ABC. Observable is a pure descriptor: it carries `name` and `indices`,
nothing else. Translation to a backend-specific form is `State`'s responsibility, just
as gate translation is `Simulator`'s. `State.expectation()` dispatches on the concrete
`Observable` subclass (`isinstance(obs, PauliString)`, etc.).

Concrete subclasses (in `core/observables.py`) add their own representation:
- `PauliString("IXIZ")` â€” takes a string; `.matrix` is a lazy `@property`
- `MatrixObservable(matrix)` â€” stores the matrix directly; serves as the universal fallback

### `NoiseModel` (ABC)

Modelled after the legacy `Noise` class.

```python
class NoiseModel(ABC):
    def get_kraus(self, gate: Gate) -> list[np.ndarray]:
        raise NotImplementedError

    def get_superop(self, gate: Gate) -> np.ndarray:
        """Implemented in terms of get_kraus; works once subclass implements get_kraus."""
        kraus = self.get_kraus(gate)
        return sum(np.kron(k.conj(), k) for k in kraus)

    def sample_error(self, gate: Gate) -> np.ndarray:
        """Not implemented here; subclasses that support sampling override this."""
        raise NotImplementedError
```

Not every subclass needs every method â€” `ThermalNoise` can provide `get_kraus` and
`get_superop` but not `sample_error`. None of these are `@abstractmethod`.

### `Simulator` (ABC)

```python
class Simulator(ABC):
    def __init__(self, circuit: Circuit, initial_state: State,
                 noise_model: NoiseModel | None = None, decompose: bool = False):
        self._validate_state(initial_state)
        self.circuit = circuit
        self.initial_state = initial_state
        self.noise_model = noise_model
        self.decompose = decompose

    def run(self) -> State:
        """Defined here; calls abstract hooks."""
        import copy
        state = copy.deepcopy(self.initial_state)
        circuit = self.circuit.copy()
        if self.decompose:
            circuit.decompose()
        circuit.construct_layers()
        circuit = self._insert_noise(circuit, self.noise_model)
        for layer in circuit.layers:
            for gate in layer:
                translated = self.translate_gate(gate)
                self._apply_translated_gate(state, translated)
        return state

    @abstractmethod
    def _validate_state(self, state: State) -> None: ...

    @abstractmethod
    def translate_gate(self, gate: Gate): ...

    @abstractmethod
    def _apply_translated_gate(self, state: State, translated_gate) -> None: ...

    def _insert_noise(self, circuit: Circuit, nm: NoiseModel | None) -> Circuit:
        """Default: raises IncompatibleNoiseError if nm is not None."""
        if nm is not None:
            raise IncompatibleNoiseError(
                f"{type(self).__name__} does not support noise models; "
                "use DensityMatrixSimulator or MonteCarloSimulator"
            )
        return circuit
```

Compile order: `decompose` (optional) â†’ `construct_layers` â†’ `_insert_noise`. Noise is
inserted after layers are determined because noise channels are inserted *between* layers.
`_insert_noise` is **not** abstract â€” the default explicitly errors if noise is passed
(fixing the v0.1 silent-drop bug). Noise-compatible backends override it.

## Complications / edge cases

- `Circuit.copy()` does a shallow copy of the gate list. Gates are treated as
  value-like (re-used across circuit copies), so deep-copying gates in `run()` is
  not needed â€” `construct_layers` only reads gate data, doesn't mutate it.
- `circuit.construct_layers()` in `run()` is called on the post-copy circuit,
  not the original. `self.circuit` is never mutated.
- `Gate.local_dim` uses `round()` to handle floating-point edge cases from the
  power operation; stored in `__init__`, not recomputed each access.

## Tests (`tests/test_core/test_base.py`)

Smoke tests â€” structural, not physics:

- `test_gate_indices_normalised`: passing `int` index gives `list[int]`
- `test_gate_dagger`: `H.dagger()` is `H` (self-inverse)
- `test_circuit_construct_layers`: 2-qubit circuit layers pack correctly
- `test_circuit_composition`: `circ + circ` doubles gate count
- `test_circuit_repetition`: `circ * 3` triples gate count
- `test_circuit_dagger`: reversed gate order, conjugate matrices
- `test_state_abstract`: can't instantiate `State` directly
- `test_simulator_abstract`: can't instantiate `Simulator` directly
- `test_simulator_run_raises_on_noise`: a minimal concrete `Simulator` subclass
  raises `IncompatibleNoiseError` when noise model passed
- `test_observable_no_methods`: `Observable` subclass with only `name`/`indices`
  works correctly; no `as_matrix` required on ABC
