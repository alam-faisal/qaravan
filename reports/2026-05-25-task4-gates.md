## Summary

Task 4 is fully implemented. All 125 tests pass.

### Changes to `core/base.py`

- `Gate` is now an ABC. `matrix` is `@property @abstractmethod`. `local_dim` is a lazy property (computed from `self.matrix` on access — needed because `ParametricGate` has no matrix at `__init__` time).
- `MatrixGate(Gate)`: concrete gate storing a fixed matrix. Use directly for ad-hoc gates.
- `ParametricGate(Gate, ABC)`: abstract gate computing its matrix from `params` via `_build_matrix()`. `__str__`/`__repr__` show `name(indices, params)` with bare name in `self.name`.
- `State.default_simulator`: `@property @abstractmethod`; each backend's `State` subclass returns its paired `Simulator` class.
- `State.apply(circuit, **kwargs)`: concrete method — calls `self.default_simulator(circuit, self, **kwargs).run()`.

### New file `core/gates.py`

**Single-qubit `MatrixGate` subclasses:** `I`, `X`, `Y`, `Z`, `H`, `S`, `Sdg`, `T`, `Tdg`, `SX`

**Single-qubit `ParametricGate` subclasses:** `RX`, `RY`, `RZ`  
Convention: `exp(-i θ P)`, full angle. `RX(0, π/2).matrix ≈ -iX`.  
`dagger()` returns same type with negated parameter.

**Two-qubit `MatrixGate` subclasses:** `CNOT`, `CZ`, `SWAP`, `iSWAP`  
CNOT now uses `(control, target)` convention (was `(target, control)` in legacy).

**Two-qubit `ParametricGate` subclasses:** `RXX`, `RYY`, `RZZ`  
Analytic expansion — no scipy dependency.  
`dagger()` returns same type with negated parameter.

**Qutrit `MatrixGate` subclasses:** `X01`, `SX01`, `X12`, `SX12`, `H01`, `SDG01`, `SWAP3`, `CNOT3`  
CNOT3 matrix derived from legacy via `SWAP3 @ M_legacy_final @ SWAP3.T` to account for convention flip.

**Utilities:**
- `random_unitary(n, local_dim=2, seed=None)` — Haar-random unitary via `eigh(A @ A†)`, uses `np.random.default_rng(seed)` (no global state mutation)
- `is_unitary(u, atol=1e-10)` — checks both `UU† ≈ I` and `U†U ≈ I`
- `pauli_string_to_gates(string)` — `"XZI"` → `[X(0), Z(1)]`, case-insensitive

## Notes

- `ParametricGate.dagger()` falls back to a frozen `MatrixGate` for gates that don't override it. All 6 rotation subclasses override it to return same-type with negated params.
