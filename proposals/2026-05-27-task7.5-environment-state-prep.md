# Proposal: Task 7.5 — Environment-based variational state preparation

## Approach

Port the environment sweep algorithm from `legacy/environments.py` into the new abstraction
layer. The core idea: for each gate `G_k`, compute the `d^k × d^k` environment matrix
`E_k = partial_overlap(pre_state, post_state, skip=indices_k)`, then replace `G_k` with its
polar factor via SVD. Maintain `pre_state` and `post_state` incrementally (one gate application
per update) to avoid full re-simulation at each step.

The design is backend-agnostic: `environment_state_prep` calls only `State.apply(Circuit([g]))`
and `State.partial_overlap(other, skip)`, both of which are on the abstract base class.

---

## Files and what changes in each

### 1. `core/base.py` — refactor `overlap` and add `State.partial_overlap`

`partial_overlap` is the one method backends implement. `overlap` becomes a concrete convenience
wrapper on the base class:

```python
def partial_overlap(self, other: State, skip: list[int]) -> np.ndarray:
    """Partial ⟨self|other⟩ keeping sites in skip uncontracted.

    Returns (local_dim^|skip| × local_dim^|skip|) matrix.
    skip=[] returns (1, 1) full overlap (use overlap() for the scalar).
    result[i, j] = ⟨other[j-block] | self[i-block]⟩.
    """
    raise NotImplementedError(
        f"{type(self).__name__} does not implement partial_overlap"
    )

def overlap(self, other: State) -> complex:
    """⟨self|other⟩. Concrete default via partial_overlap; backends need not override."""
    return self.partial_overlap(other, skip=[])[0, 0]
```

`overlap` is removed from the list of abstract methods in `State`. Backends that previously
implemented `overlap` directly (e.g. `Statevector`) can drop that override — they get it free.

Non-abstract with `raise NotImplementedError` — same pattern as `NoiseModel.get_kraus`.

### 2. `backends/statevector.py` — implement `Statevector.partial_overlap`, drop `overlap`

```python
def partial_overlap(self, other: Statevector, skip: list[int]) -> np.ndarray:
    """Wraps module-level partial_overlap; result[i,j] = ⟨other[j]|self[i]⟩ in skip subspace."""
    # module-level partial_overlap is in scope via global lookup, not self
    return partial_overlap(self.to_array(), other.to_array(), local_dim=self.local_dim, skip=skip)
```

Remove `Statevector.overlap` — the base class concrete method handles it.

Consistency check: `sv.partial_overlap(sv, skip=sites)` must equal `sv.rdm(sites)`. Both call
`partial_overlap(flat, flat, skip=sites)` so this holds by construction.

### 3. `applications/circuit_library.py` — add `brickwall_skeleton` (2D) and `two_local_circuit`

Port of the Julia 2D brickwall. One pass = even-x horizontals, odd-x horizontals, even-y
verticals, odd-y verticals. For `Ly=1` this reduces to the standard 1D brickwall.

```python
def brickwall_skeleton(Lx: int, Ly: int = 1) -> list[list[int]]:
    """NN edges in brickwall order for Lx×Ly grid (row-major, 0-indexed).

    Returns edge list: one pass = even-x horiz, odd-x horiz, even-y vert, odd-y vert.
    For depth-d circuits pass the result repeated: brickwall_skeleton(Lx, Ly) * d.
    """
    h_edges, v_edges = [], []
    for y in range(Ly):
        for x in range(Lx):
            a = y * Lx + x
            if x < Lx - 1:
                h_edges.append((a, a + 1))
            if y < Ly - 1:
                v_edges.append((a, a + Lx))

    skeleton: list[list[int]] = []
    skeleton.extend([list(e) for e in h_edges if (e[0] % Lx) % 2 == 0])
    skeleton.extend([list(e) for e in h_edges if (e[0] % Lx) % 2 == 1])
    skeleton.extend([list(e) for e in v_edges if (e[0] // Lx) % 2 == 0])
    skeleton.extend([list(e) for e in v_edges if (e[0] // Lx) % 2 == 1])
    return skeleton


def two_local_circuit(skeleton: list[list[int]], seed: int | None = None) -> Circuit:
    """Circuit of Haar-random 4×4 unitaries on each pair in skeleton."""
    rng = np.random.default_rng(seed)
    gates = [
        MatrixGate(f"U{i}", indices, random_unitary(len(indices), seed=int(rng.integers(2**31))))
        for i, indices in enumerate(skeleton)
    ]
    n = max(idx for pair in skeleton for idx in pair) + 1
    return Circuit(gates, num_sites=n)
```

`random_unitary` is already in `core/gates.py`; add it to the import line in `circuit_library.py`.

### 4. `applications/run_context.py` — new file

```python
from __future__ import annotations
from dataclasses import dataclass, field
import numpy as np


@dataclass
class RunContext:
    """Stopping criteria and bookkeeping for iterative optimization loops.

    Used by environment_state_prep and any future iterative algorithm.
    max_iter:       hard cap on number of full sweeps.
    stop_ratio:     stop if |Δcost| / |cost| < stop_ratio for a full sweep.
    stop_absolute:  stop if cost < stop_absolute.
    quiet:          suppress per-sweep log lines.
    """
    max_iter: int = 1000
    stop_ratio: float = 1e-6
    stop_absolute: float | None = None
    quiet: bool = True

    def should_stop(self, sweep_costs: list[float], sweep: int) -> bool:
        """Return True if any stopping criterion is met. sweep_costs is one entry per sweep."""
        if sweep >= self.max_iter:
            return True
        if self.stop_absolute is not None and sweep_costs[-1] < self.stop_absolute:
            return True
        if self.stop_ratio is not None and len(sweep_costs) >= 2:
            delta = abs(sweep_costs[-1] - sweep_costs[-2])
            if delta < self.stop_ratio * (abs(sweep_costs[-2]) + 1e-12):
                return True
        return False
```

`should_stop` receives per-sweep costs (one entry per complete left+right pass), not per-gate
costs. This keeps convergence checking unambiguous. The returned `cost_list` from
`environment_state_prep` is per-gate (finer granularity); the function tracks sweep-end indices
internally.

### 5. `applications/compilation.py` — new file

Public API: `environment_state_prep`.
Private helper: `_environment_update` (module-level, prefixed `_`).

```python
def environment_state_prep(
    target: State,
    init_state: State,
    circuit: Circuit | None = None,
    skeleton: list[list[int]] | None = None,
    context: RunContext | None = None,
) -> tuple[Circuit, list[float]]:
    """Optimise a circuit to prepare target from init_state via the environment sweep.

    Exactly one of circuit or skeleton must be provided.
    Returns (optimised_circuit, cost_list) where cost is infidelity 1 - |⟨target|U|init⟩|
    recorded after every gate update (smooth convergence curve).
    Input circuit is never mutated — the function works on a copy.
    """
```

```python
def _environment_update(
    circ: Circuit,
    k: int,
    pre_state: State,
    post_state: State,
    init_state: State,
    target: State,
    direction: str,          # "left" (decreasing k) or "right" (increasing k)
) -> tuple[float, State, State]:
    """Update gate k in-place and return (cost, new_pre_state, new_post_state).

    circ is owned by the caller (already a copy); mutating circ.gates[k] is intentional.
    """
```

**Circuit mutation pattern**: `circ = circuit.copy()` at the top of `environment_state_prep`
(same as `Simulator.run()`), then `circ.gates[k] = MatrixGate(circ.gates[k].name, circ.gates[k].indices, new_mat)`
inside `_environment_update`. `circ.layers = None` after each gate replacement (invalidate
the layer cache). No `set_gate` method on `Circuit`.

**Single-gate circuit in incremental updates**: `state.apply(Circuit([gate]))` — no `num_sites`
argument. The gate carries its own `indices`; the state carries its own `num_sites`; the
simulator reads both. Passing `num_sites` would be redundant.

**Incremental state updates**: ported faithfully from `legacy/environments.py`. For direction
`"left"` (sweep k from n−1 down to 1):
- After updating gate k: `pre_state` ← undo gate k−1 (apply `gate[k-1].dagger()`); `post_state` ← apply updated `gate[k]†`.
- Special case k=1: `pre_state` ← `init_state` directly.

For direction `"right"` (sweep k from 0 up to n−2):
- After updating gate k: `pre_state` ← apply updated `gate[k]`; `post_state` ← undo gate k+1 (apply `gate[k+1]`).
- Special case k=n−2: `post_state` ← `target` directly.

State evolution uses `state.apply(Circuit([gate]))` — already available on all `State` subclasses.

**Initialisation of pre_state / post_state** before the first sweep:
```python
# Run the full circuit once to get the initial ansatz
ansatz = init_state.apply(circ)
cost_list = [1 - abs(target.overlap(ansatz))]

# pre_state: full evolution minus the last gate (undo it)
pre_state = ansatz.apply(Circuit([circ.gates[-1].dagger()]))
post_state = target
```

**Full sweep structure** (per-gate cost recording):
```python
sweep_costs: list[float] = []

for sweep in range(context.max_iter):
    # left sweep: k from len(circ)-1 down to 1
    for k in reversed(range(1, len(circ))):
        cost, pre_state, post_state = _environment_update(circ, k, pre_state, post_state, init_state, target, "left")
        cost_list.append(cost)

    # right sweep: k from 0 up to len(circ)-2
    for k in range(len(circ) - 1):
        cost, pre_state, post_state = _environment_update(circ, k, pre_state, post_state, init_state, target, "right")
        cost_list.append(cost)

    sweep_costs.append(cost_list[-1])   # sweep-end cost for convergence check
    if context.should_stop(sweep_costs, sweep + 1):
        break

return circ, cost_list
```

## Edge cases and complications

**Naming conflict inside `Statevector.partial_overlap`**: calling the bare name `partial_overlap()`
inside the method body resolves to the module-level function (Python's scoping looks in global
namespace, not class namespace). No alias needed; a comment notes this.

**`random_unitary` in `two_local_circuit`**: each gate gets an independent seed drawn from the
top-level RNG via `rng.integers(2**31)`, preserving global reproducibility without changing the
`random_unitary` signature.

**Skeleton with non-nearest-neighbour pairs**: `two_local_circuit` accepts any `list[list[int]]`;
`brickwall_skeleton` is one factory for standard hardware topologies. `num_sites` in the resulting
circuit is inferred as `max(all indices) + 1`, so non-contiguous qubit indices work correctly.

**Zero-cost initialisation**: if `target.overlap(init_state.apply(circ))` is already 1.0 (or
very close), the first sweep will still run but all SVDs will immediately give optimal gates.
This is correct and harmless.

**`overlap` on `Statevector` drops its override**: since `overlap` is now concrete on `State`,
`Statevector.overlap` can be removed. All callers hit the base-class method which calls
`self.partial_overlap(other, skip=[])[0,0]`. Same result, one less override.

```python
def test_partial_overlap_cross_state():
    sv1 = Statevector(3, random_seed=1)
    sv2 = Statevector(3, random_seed=2)
    po = sv1.partial_overlap(sv2, skip=[1])
    # shape check
    assert po.shape == (2, 2)
    # manual check: contract sites 0 and 2, leave site 1 free
    # result[i,j] = sum_{a,b} sv1[a,i,b].conj() * sv2[a,j,b]
    t1 = sv1._tensor   # shape (2,2,2)
    t2 = sv2._tensor
    manual = np.einsum("aib,ajb->ij", t1.conj(), t2)
    np.testing.assert_allclose(po, manual, atol=1e-12)
```
